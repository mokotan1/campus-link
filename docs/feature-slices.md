# Campus Link 기능 슬라이스

이 문서는 3주 MVP를 한 번에 한 개발자 또는 AI 에이전트가 처리할 수 있는 크기의 기능 슬라이스로 나눈 목록입니다.

## 1주차

### 슬라이스 1. 프로젝트 골격

바운디드 컨텍스트: Common

목표: 프론트엔드·백엔드 기본 프로젝트를 만든다.

담당: 프론트엔드 + 백엔드

완료 조건:

- Next.js 앱이 로컬에서 실행된다.
- Spring Boot 앱이 로컬에서 실행된다.
- README에 로컬 실행 명령이 있다.

### 슬라이스 2. 공통 도메인 상수

바운디드 컨텍스트: Common

목표: 캠퍼스, 역할, 프로젝트 유형, 프로젝트 상태, 지원 상태 값을 정의한다.

담당: PM + 백엔드 + 프론트엔드

완료 조건:

- 프론트엔드와 백엔드가 같은 이름을 사용한다.
- 더미 데이터가 같은 값을 사용한다.

### 슬라이스 3. 온보딩 기본 정보

바운디드 컨텍스트: Profile

목표: 사용자가 캠퍼스, 학과, 학년, 학교 이메일을 입력한다.

담당: 프론트엔드

완료 조건:

- 온보딩 1단계가 프로토타입과 일치한다.
- 입력값이 검증된다.

## 2주차

### 슬라이스 4. 역할 선택

바운디드 컨텍스트: Profile

목표: 사용자가 역할 태그와 툴 태그를 선택한다.

담당: 프론트엔드

완료 조건:

- 여러 역할을 선택할 수 있다.
- 툴 입력이 로컬/목(mock) 상태에 저장된다.

### 슬라이스 5. 포트폴리오 등록

바운디드 컨텍스트: Portfolio

목표: 사용자가 링크와 역할 설명이 포함된 포트폴리오 항목을 등록한다.

담당: 프론트엔드 + 백엔드

완료 조건:

- 포트폴리오 입력 폼이 있다.
- 포트폴리오 API 계약이 정의되어 있다.

### 슬라이스 6. 프로젝트 등록

바운디드 컨텍스트: Project

목표: 사용자가 프로젝트 모집 글을 작성한다.

담당: 백엔드 + 프론트엔드

완료 조건:

- 프로젝트 생성 API가 있다.
- 프로젝트 등록 화면이 있다.

### 슬라이스 7. 프로젝트 목록 및 필터

바운디드 컨텍스트: Project

목표: 사용자가 프로젝트를 탐색하고 필터링할 수 있다.

담당: 프론트엔드

완료 조건:

- 검색, 캠퍼스, 역할, 상태 필터가 더미 데이터로 동작한다.

## 3주차

### 슬라이스 8. 프로젝트 지원

바운디드 컨텍스트: Application

목표: 사용자가 프로젝트에 지원한다.

담당: 백엔드 + 프론트엔드

완료 조건:

- 지원 버튼으로 지원(application)이 생성된다.
- 지원 상태가 pending(대기)으로 시작한다.

### 슬라이스 9. 인재에게 제안

바운디드 컨텍스트: Application

목표: 프로젝트 소유자가 사용자에게 협업을 제안한다.

담당: 백엔드 + 프론트엔드

완료 조건:

- 제안(proposal) 액션이 있다.
- 제안이 지원 상태 화면에 표시된다.

### 슬라이스 10. 지원 현황 페이지

바운디드 컨텍스트: Application

목표: 사용자가 보낸 지원과 받은 제안을 확인한다.

담당: 프론트엔드

완료 조건:

- 대기, 수락, 거절, 취소 상태가 보인다.

### 슬라이스 11. 데모 데이터 및 QA

바운디드 컨텍스트: Common

목표: 데모 가능한 데이터를 준비하고 전체 흐름을 테스트한다.

담당: 테스트/데이터

완료 조건:

- 프로젝트가 최소 10개 있다.
- 포트폴리오가 최소 10개 있다.
- 데모 시나리오를 오류 없이 끝까지 진행할 수 있다.
