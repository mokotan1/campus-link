# Campus Link 기능 스펙 및 기술 스택

## 1. 서비스 개요

Campus Link는 대명캠의 아트/영상/애니메이션 인력과 성서캠의 게임 개발/기획 인력을 연결하는 학교 기반 협업 매칭 웹 서비스다.

기존 에브리타임 게시판은 사람을 찾을 수는 있지만, 팀의 준비 상태나 개인 작업물의 퀄리티를 확인하기 어렵다. Campus Link는 회원가입 직후 온보딩을 통해 캠퍼스, 역할, 포트폴리오, 협업 가능 상태를 구조화하고, 이를 기준으로 프로젝트 추천과 지원/제안 흐름을 만든다.

## 2. 현재 HTML UI 시안

현재 HTML 프로토타입 파일:

`outputs/campus-link-ui.html`

현재 UI 구성:

- 온보딩 화면
- 기본 정보 입력
- 역할 선택
- 포트폴리오 등록
- 협업 가능 상태 선택
- 맞춤 추천 결과
- 추천 프로젝트 대시보드
- 프로젝트 카드
- 추천 인재 카드
- 지원 현황 패널

## 3. 핵심 사용자

### 대명캠 사용자

- 영상애니메이션, 시각디자인, 2D/3D 아트, 캐릭터 디자인, UI, 애니메이션 작업자
- 개발자가 필요한 졸업작품/포트폴리오 팀
- 자신의 작업물을 게임 프로젝트에 적용하고 싶은 사용자

### 성서캠 사용자

- 게임 개발자, 프론트엔드 개발자, Unity/Unreal/Godot 사용자
- 아트/애니메이션/사운드 인력이 필요한 팀
- 졸업작품, 공모전, 게임잼, 포트폴리오 프로젝트를 준비하는 사용자

## 4. MVP 기능

### 4.1 회원가입 및 로그인

- 학교 이메일 회원가입
- 이메일 인증
- 로그인/로그아웃
- 비밀번호 재설정
- 사용자 기본 프로필 생성

### 4.2 온보딩

- 캠퍼스 선택: 대명캠, 성서캠
- 학과, 학년 입력
- 대표 역할 선택
- 보조 역할 선택
- 사용 가능 툴 입력
- 협업 가능 상태 선택
- 원하는 협업 유형 선택
- 주당 가능 시간 입력

### 4.3 포트폴리오 프로필

- 대표 작업물 업로드
- 이미지, 영상 썸네일, 외부 링크 등록
- 작업물별 본인 역할 입력
- 사용 툴 태그
- 협업 가능 여부 표시
- 캠퍼스/전공/역할 기반 프로필 노출

### 4.4 프로젝트 등록

- 프로젝트 제목
- 프로젝트 설명
- 프로젝트 유형: 졸업작품, 게임잼, 공모전, 단기 포트폴리오, 장기 프로젝트
- 현재 팀 구성
- 모집 중인 역할
- 필요 툴 또는 기술
- 기획서/시안/프로토타입 링크
- 모집 상태: 모집중, 진행중, 완료

### 4.5 프로젝트 탐색

- 프로젝트 목록
- 검색
- 캠퍼스 필터
- 역할 필터
- 프로젝트 상태 필터
- 협업 유형 필터
- 프로젝트 상세 페이지

### 4.6 지원하기/제안하기

- 프로젝트에 지원하기
- 인재에게 제안하기
- 지원 메시지 작성
- 포트폴리오 자동 첨부
- 지원 상태 확인: 대기, 수락, 거절, 취소

### 4.7 지원 현황

- 내가 지원한 프로젝트 목록
- 내가 받은 제안 목록
- 상태별 분류
- 프로젝트 담당자와 연결

- 플레이 링크 또는 영상 링크
- 졸업작품/공모전 결과 기록

### 5.5 관리자 기능

- 사용자 신고 관리
- 부적절한 게시글 숨김

- 프로젝트/포트폴리오 관리

## 6. 추천 기술 스택

## 6.1 프론트엔드

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- Lucide Icons
- React Hook Form
- Zod

선택 이유:

- Next.js는 페이지 라우팅, 서버 기능, API 처리, 배포 흐름이 좋아서 풀스택 웹서비스에 적합하다.
- TypeScript는 사용자, 프로젝트, 지원 상태 같은 데이터 구조를 안정적으로 관리하기 좋다.
- Tailwind CSS와 shadcn/ui는 빠르게 깔끔한 UI를 만들기에 좋다.
- React Hook Form과 Zod는 온보딩처럼 입력 단계가 많은 화면에서 유용하다.

## 6.2 백엔드 및 데이터베이스

- Supabase
- PostgreSQL
- Supabase Auth
- Supabase Storage
- Supabase Realtime
- Supabase Row Level Security

선택 이유:

- 회원가입, 로그인, DB, 파일 업로드를 빠르게 붙일 수 있다.
- 포트폴리오 이미지와 영상 파일을 Storage로 관리할 수 있다.
- PostgreSQL 기반이라 역할, 프로젝트, 지원 내역처럼 관계가 많은 데이터를 다루기 좋다.
- 나중에 채팅이나 알림 기능이 필요할 때 Realtime을 활용할 수 있다.

## 6.3 배포

- Vercel
- Supabase Cloud

선택 이유:

- Next.js와 Vercel의 궁합이 좋다.
- 프론트엔드와 백엔드를 빠르게 배포할 수 있다.
- 학교 프로젝트나 졸업작품 수준에서 관리 부담이 낮다.

## 6.4 개발 도구

- GitHub
- ESLint
- Prettier
- Figma
- Notion

선택 이유:

- GitHub는 협업과 버전 관리에 필요하다.
- ESLint/Prettier는 코드 스타일을 유지한다.
- Figma는 UI 설계와 발표 자료 제작에 좋다.
- Notion은 기획서, 기능 명세, 프로젝트 진행 상황 관리에 적합하다.

## 10. 발표용 핵심 차별점

- 에브리타임은 게시글 중심, Campus Link는 검증된 프로필 중심
- 단순 모집 게시판이 아니라 프로젝트 상태와 작업물을 함께 확인
- 캠퍼스 간 물리적 거리 문제를 온보딩과 추천 구조로 해결
- 지원하기/제안하기 이후 지원 현황까지 이어지는 명확한 협업 흐름
- 완성작 아카이브를 통해 졸업작품과 포트폴리오 퀄리티 향상에 기여
