## 제품 한 줄 정의

Campus Link는 대명캠의 아트, 영상, 애니메이션 인력과 성서캠의 개발, 기획 인력을 연결하는 학교 기반 협업 매칭 웹서비스입니다.

핵심 차별점은 단순 모집 게시판이 아니라, 검증 가능한 프로필과 준비된 프로젝트 정보를 바탕으로 팀을 찾고 지원 또는 제안을 주고받게 하는 것입니다.

## MVP 성공 기준

- 사용자가 학교 이메일로 가입하고 온보딩을 완료할 수 있습니다.
- 사용자가 외부 포트폴리오 링크와 본인 역할을 등록할 수 있습니다.
- 사용자가 프로젝트를 등록하고 목록에서 검색/필터로 탐색할 수 있습니다.
- 사용자가 프로젝트에 지원하거나, 프로젝트 소유자가 인재에게 제안할 수 있습니다.
- 프로젝트 소유자가 받은 지원을 수락/거절할 수 있습니다.
- 지원 또는 제안이 수락되면 양쪽 사용자에게 연락 수단이 공개됩니다.
- 더미데이터로 전체 시연 흐름을 3분 안에 끊김 없이 보여줄 수 있습니다.

## MVP 핵심 정책

- 실시간 채팅과 푸시 알림은 제외합니다.
- 파일 업로드는 제외하고 외부 포트폴리오 링크만 등록합니다.
- 추천 기능은 P1로 두고, 목록/필터/지원/제안 흐름을 P0로 둡니다.
- 검증된 프로필은 실력 보증이 아니라 프로필 준비도 배지로 정의합니다.

## 기능 우선순위

### P0 - 시연 필수

- 회원가입/로그인
- 온보딩 정보 저장
- 외부 포트폴리오 링크 등록
- 프로젝트 등록/목록/상세
- 검색/필터
- 지원하기/제안하기
- 받은 지원/받은 제안 조회
- 수락/거절 상태 변경
- 수락 후 연락 수단 공개

### P1 - 있으면 좋은 기능

- 추천 프로젝트
- 추천 인재
- 핵심 상황 이메일 알림
- 프로필 준비도 배지

### P2 - 확장 기능

- 파일 업로드
- 실시간 채팅
- 푸시 알림
- 관리자 페이지
- 후기/평점
- 정교한 추천 알고리즘

## 사용자 흐름

1. 사용자가 학교 이메일로 가입합니다.
2. 온보딩에서 캠퍼스, 학과, 학년을 입력합니다.
3. 프로젝트에서 맡을 수 있는 역할과 사용 가능한 툴을 선택합니다.
4. 외부 포트폴리오 링크와 작업물 내 본인 역할을 등록합니다.
5. 협업 가능 상태와 원하는 프로젝트 유형을 입력합니다.
6. 입력 정보를 기반으로 프로젝트 목록과 인재 목록을 확인합니다.
7. 프로젝트 상세를 보고 지원하거나, 프로젝트 소유자가 인재에게 제안합니다.
8. 지원 현황에서 보낸 지원, 받은 지원, 받은 제안 상태를 확인합니다.
9. 프로젝트 소유자는 받은 지원을 수락하거나 거절합니다.
10. 사용자는 받은 제안을 수락하거나 거절합니다.
11. 지원 또는 제안이 수락되면 양쪽 사용자에게 연락 수단이 공개됩니다.

## 온보딩 화면 의도

### 1단계 - 기본 정보

입력 항목은 이름/닉네임, 캠퍼스, 학과, 학년, 학교 이메일입니다.

캠퍼스와 학과는 단순 프로필 정보가 아니라 추천과 매칭에 쓰이는 기본 데이터입니다. 학교 이메일은 외부 사용자의 무분별한 진입을 줄이고 학교 기반 협업 플랫폼이라는 신뢰를 만드는 장치입니다.

### 2단계 - 역할 선택

개발, 기획, 2D 아트, 3D 모델링, 애니메이션, UI/UX 역할을 복수 선택할 수 있어야 합니다. 사용 가능 툴도 입력합니다.

역할과 툴은 프로젝트 수요와 인재 역량을 구조화하는 핵심 태그입니다.

### 3단계 - 작업물 검증

외부 포트폴리오 링크, 대표 이미지 또는 영상 썸네일 URL, 작업물 내 본인 역할을 입력합니다.

MVP는 파일 업로드 대신 ArtStation, GitHub, YouTube, Behance, Notion, Google Drive 같은 외부 링크 등록으로 시작합니다. 팀 프로젝트 결과물은 본인이 맡은 역할을 별도로 입력해야 합니다.

### 4단계 - 협업 상태

협업 가능 상태는 바로 가능, 일정 맞으면 가능, 구경만, 팀 보유 중으로 나뉩니다. 원하는 협업 유형과 주당 가능 시간도 받습니다.

이 정보는 매칭 이후 연락 과정의 시간 낭비를 줄이는 기준입니다.

### 5단계 - 맞춤 추천

온보딩이 끝나면 추천 결과 준비 완료 카드, 추천 태그, 대시보드 보기 버튼을 보여줍니다.

MVP 추천은 정교한 알고리즘보다 역할, 캠퍼스, 프로젝트 상태, 툴, 협업 가능 상태 기반의 태그 추천으로 시작합니다.

## 대시보드와 주요 카드

대시보드는 사용자가 실제로 팀을 찾고 행동하는 중심 화면입니다.

필수 구성:

- 프로젝트 검색
- 캠퍼스 필터
- 역할 필터
- 모집 상태 필터
- 프로젝트 등록 버튼
- 팀원 모집 카드
- 추천 인재 카드
- 내 지원 현황 카드

프로젝트 카드는 썸네일, 제목, 설명, 태그, 검증 상태, 지원하기/제안하기 버튼을 포함합니다.

추천 인재 카드는 이름/닉네임, 캠퍼스, 역할, 사용 툴, 협업 가능 상태를 보여줍니다.

지원 현황은 내가 지원한 프로젝트, 내가 받은 제안, 상태 확인을 포함합니다. 상태값은 대기, 수락, 거절, 취소입니다.

## 검증된 프로필 정의

MVP에서 검증된 프로필은 학교가 실력이나 작업 품질을 공식 보증한다는 뜻이 아닙니다. 협업 판단에 필요한 정보를 충분히 입력했다는 프로필 준비도 배지입니다.

프로필 준비도 배지 기준:

- 학교 이메일 인증 완료
- 캠퍼스, 학과, 역할 태그 입력 완료
- 공개 포트폴리오 또는 외부 작업물 링크 1개 이상 등록
- 작업물 내 본인 역할 입력
- 협업 가능 상태 입력

## 보안 기준

Notion 기획서는 아래 보안 기준을 MVP 필수로 둡니다.

- Supabase Auth 기반 학교 이메일 인증
- 사용자별 프로필/프로젝트/지원 내역 권한 검증
- 주요 테이블 Row Level Security 적용
- Next.js 서버 로직과 DB 제약 조건 기반 입력값 검증
- HTTPS 사용
- 포트폴리오 외부 링크 URL 검증
- 환경변수와 API Key Git 커밋 금지
- Supabase Service Role Key 브라우저 노출 금지
- 개인정보와 인증 토큰 로그 기록 금지

외부 링크는 서버에서 URL 형식을 검증하고, 허용하지 않는 프로토콜을 차단하며, 새 창으로 열 때 `rel="noopener noreferrer"`를 적용합니다.

## 기술 스택 관련 주의

Notion 원본은 MVP 기술 스택을 `Next.js + Supabase`로 정의하고, 별도 Spring Boot 서버는 MVP 범위에서 제외한다고 적고 있습니다.

반면 현재 repo 문서에는 Spring Boot 백엔드와 DDD 기반 모듈러 모놀리스 규칙도 존재합니다. 이 둘이 충돌하는 작업에서는 임의로 판단하지 말고, 작업 요청 또는 최신 Notion 하위 명세서를 기준으로 확인한 뒤 진행합니다.
